🚀 Introduction

Noobly JS is a modern, lightweight accelerator framework designed to help developers build robust web applications quickly and efficiently. Built with performance and developer experience in mind, it provides a clean, intuitive API that doesn't get in your way.

Why Noobly JS?

⚡

Fast by Default

Optimized for performance with minimal overhead and intelligent caching strategies.

🧩

Modular Design

Use only what you need with our plugin-based architecture and tree-shaking support.

🛠️

Developer Friendly

Intuitive APIs, excellent TypeScript support, and comprehensive documentation.

💡 Pro Tip: Noobly JS works great with existing Node.js applications. You can gradually migrate or start fresh!

📦 Installation

Getting started with Noobly JS is quick and easy. Choose your preferred package manager and follow the steps below.

Prerequisites

  • Node.js 16 or higher
  • npm, yarn, or pnpm

Install Noobly JS

Package Manager Options

npm
yarn
pnpm
npm install noobly-js
yarn add noobly-js
pnpm add noobly-js
✅ Success! Noobly JS is now installed and ready to use in your project.

🏃‍♂️ Quick Start

Let's create your first Noobly JS application in just a few minutes!

  1. Create a new project

    Initialize a new Node.js project and install Noobly JS:

    mkdir my-noobly-app
cd my-noobly-app
npm init -y
npm install noobly-js

  2. Create your app.js file

    Set up a basic server with routing:

    const { NooblyApp } = require('noobly-js');

const app = new NooblyApp();

app.get('/', (req, res) => {
  res.json({ message: 'Hello from Noobly JS!' });
});

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});

  3. Start your application

    Run your application and visit http://localhost:3000:

    node app.js
🎉 Congratulations! You've just created your first Noobly JS application. Check out the core concepts to learn more.

⚙️ Configuration

Noobly JS provides flexible configuration options to customize your application behavior. Configuration can be set through environment variables, config files, or programmatically.

Basic Configuration

const app = new NooblyApp({
  port: 3000,
  cors: true,
  logging: 'combined',
  database: {
    url: process.env.DATABASE_URL,
    options: {
      poolSize: 10,
      useNewUrlParser: true
    }
  },
  cache: {
    provider: 'redis',
    url: process.env.REDIS_URL
  }
});

Environment Variables

Create a .env file in your project root:

PORT=3000
NODE_ENV=development
DATABASE_URL=mongodb://localhost:27017/myapp
REDIS_URL=redis://localhost:6379
JWT_SECRET=your-secret-key
⚠️ Security Warning: Never commit your .env file to version control. Add it to your .gitignore file.

🛣️ Routing

Noobly JS provides a powerful and flexible routing system that supports all HTTP methods, middleware, and parameter validation.

Basic Routes

// GET route
app.get('/users', (req, res) => {
  res.json({ users: [] });
});

// POST route
app.post('/users', (req, res) => {
  const user = req.body;
  // Create user logic
  res.status(201).json(user);
});

// Route with parameters
app.get('/users/:id', (req, res) => {
  const { id } = req.params;
  res.json({ user: { id } });
});

// Route with query parameters
app.get('/search', (req, res) => {
  const { q, page = 1 } = req.query;
  res.json({ query: q, page });
});

Route Groups

// Group routes with common prefix
app.group('/api/v1', (router) => {
  router.get('/users', getUsersController);
  router.post('/users', createUserController);
  router.put('/users/:id', updateUserController);
  router.delete('/users/:id', deleteUserController);
});

HTTP Methods

  • app.get(path, handler) - Handle GET requests
  • app.post(path, handler) - Handle POST requests
  • app.put(path, handler) - Handle PUT requests
  • app.delete(path, handler) - Handle DELETE requests
  • app.patch(path, handler) - Handle PATCH requests
  • app.all(path, handler) - Handle all HTTP methods
